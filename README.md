customer-support-ai-agent-prompt-qa

## Project Overview

This is a portfolio project focused on designing and testing prompts for a fictional e-commerce customer-support AI agent.

The goal of this project is to demonstrate practical Prompt Engineering skills, including prompt design, intent handling, conversation-flow planning, edge-case testing, guardrail definition, response evaluation, and prompt iteration.

This project does **not** use paid APIs, production customer data, or real company systems. It is a manual prompt QA project built using ChatGPT, documentation, and structured test cases.

## Role Simulated

In this project, I acted as a beginner Prompt Engineer and AI Agent QA tester responsible for:

* Defining the AI agent’s purpose and support scope
* Writing a system prompt for the customer-support agent
* Mapping expected conversation flows
* Creating guardrails and escalation rules
* Designing test cases across common and difficult customer situations
* Evaluating AI responses for quality, safety, clarity, and policy alignment
* Improving prompts based on observed response failures
* Documenting the work in a way that can be explained during interviews

## Fictional Business Context

The project assumes a fictional e-commerce company that sells consumer products online.

The AI agent is designed to assist customers with common support requests such as:

1. Order status queries
2. Refund requests
3. Cancellation requests
4. Damaged item complaints
5. Wrong item complaints
6. Missing item complaints
7. Payment disputes
8. Fraud or scam complaints
9. Angry or frustrated customers
10. Human escalation requests

The agent is not expected to access real order databases, process refunds, cancel orders directly, or make legal/financial decisions. Instead, it should collect relevant details, explain general next steps, set expectations, and escalate when needed.

## Project Objective

The objective is to create a realistic prompt QA workflow for a customer-support AI agent.

The project focuses on answering these questions:

* Can the AI understand the customer’s intent?
* Can it respond clearly and professionally?
* Can it avoid making unsupported claims?
* Can it ask for missing information when needed?
* Can it identify situations that require human escalation?
* Can it handle angry or sensitive users without becoming defensive?
* Can it follow refund, cancellation, fraud, and complaint-handling rules?
* Can the prompt be improved after testing failures?
